## Структура модуля (папка /ml)

Мы разделили процесс обучения на логические этапы:

* **`train_rf.py`**: Скрипт для обучения модели на основе алгоритма *Random Forest*. Сохраняет обученную модель в `model_rf.pkl`.
* **`train_lr.py`**: Скрипт для обучения модели на основе *Логистической регрессии*. Сохраняет модель в `model_lr.pkl`.
* **`evaluate_and_select.py`**: Скрипт-"судья". Загружает обе модели, сравнивает их качество на тестовой выборке по метрике ROC-AUC и выбирает лучшую.
* **`model.pkl`**: Финальная (лучшая) модель, которая используется бэкендом для реальных предсказаний.
* **`loan_data.csv`**: Исходный датасет с данными заемщиков.

## Ход работы
1. **Предобработка**: В каждом скрипте реализован пайплайн (Pipeline), включающий `StandardScaler` для нормализации данных и `LabelEncoder` для обработки категориальных признаков.
2. **Обучение**: Запуск независимых скриптов обучения позволяет гибко настраивать гиперпараметры для каждого алгоритма.
3. **Оценка и выбор**: Скрипт-судья автоматически проводит валидацию моделей и обновляет файл `model.pkl`, гарантируя, что на сайте всегда работает самый точный алгоритм.
