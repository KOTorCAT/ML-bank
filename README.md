# Сервис одобрения ипотеки

Сервис на основе машинного обучения, который предсказывает, одобрят клиенту ипотеку или нет. Состоит из серверной части на Python (FastAPI) и веб-интерфейса (HTML/CSS/JavaScript).

## Что делает сервис

Вы вводите данные клиента: возраст, доход, сумма кредита, кредитную историю и другие параметры. Сервис возвращает прогноз — одобрят ипотеку или откажут.

Работает тремя способами:

1. Через веб-интерфейс в браузере — заполняете форму, получаете результат
2. Через загрузку CSV-файла — загружаете файл с данными нескольких клиентов, сервис обрабатывает всех сразу
3. Через API — отправляете JSON-запрос из своей программы или из Postman

## Как запустить проект


### Пошаговая инструкция запуска


**Шаг  Создайте виртуальное окружение**

Виртуальное окружение — это изолированная среда для библиотек Python, чтобы они не пересекались с другими проектами. Создаётся один раз.

```
python -m venv venv
```

Появится папка `venv` внутри проекта.

**Шаг 2. Активируйте виртуальное окружение**

На Windows (командная строка или PowerShell):

```
venv\Scripts\activate
```

На Mac или Linux:

```
source venv/bin/activate
```

После активации в начале строки терминала появится `(venv)`. Это знак, что окружение активно. Его нужно активировать каждый раз перед запуском сервера.

**Шаг 3. Установите зависимости**

```
pip install -r requirements.txt
```

Подождите минуту-две. Pip скачает и установит все нужные библиотеки: FastAPI, uvicorn, pandas, scikit-learn и другие.

**Шаг 4. Запустите сервер**

```
python -m app.main
```

Если всё работает, вы увидите в терминале примерно такие строки:

```
INFO:     Started server process
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

Сервер запущен и готов принимать запросы.

**Шаг 5. Откройте веб-интерфейс**

Откройте браузер и перейдите по адресу:

```
http://localhost:8000/app
```

Откроется страница сервиса. Если модель ещё не загружена, в правом верхнем углу будет написано "Модель не загружена".

## Как пользоваться

### Способ 1. Ручной ввод данных через форму

1. Откройте веб-интерфейс: `http://localhost:8000/app`
2. Заполните поля формы данными клиента. Пример заполнения:
   - Возраст: 25
   - Пол: Мужской
   - Образование: Высшее образование
   - Доход: 50000
   - Стаж работы: 5
   - Тип жилья: Собственное
   - Сумма кредита: 150000
   - Цель кредита: Целевое
   - Процентная ставка: 6.5
   - Отношение кредита к доходу: 0.3
   - Кредитная история: 5
   - Кредитный скоринг: 720
   - Дефолты в прошлом: Да
3. Нажмите кнопку "Добавить запись". Запись появится в таблице ниже формы.
4. Можно добавить несколько записей.
5. Нажмите "Отправить все записи на предсказание". Сервис обработает все записи и покажет результат для каждой: "Одобрено" или "Отказ", а также вероятность.

### Способ 2. Загрузка CSV-файла

1. Подготовьте CSV-файл со следующими колонками (именно в таком порядке и с такими названиями):
   ```
   person_age,person_gender,person_education,person_income,person_emp_exp,person_home_ownership,loan_amnt,loan_intent,loan_int_rate,loan_percent_income,cb_person_cred_hist_length,credit_score,previous_loan_defaults_on_file
   ```
2. Каждая следующая строка — данные одного клиента. Значения разделяются запятыми.
3. В веб-интерфейсе найдите секцию "Предсказание из CSV-файла".
4. Нажмите "Выберите файл" и выберите ваш CSV.
5. Нажмите "Обработать CSV". Сервис вернёт результаты с предсказаниями.

### Способ 3. Через API

Если вы разработчик и хотите использовать сервис из своего кода, используйте эндпоинты ниже.

## API эндпоинты

Сервер работает по адресу `http://localhost:8000`, если запущен локально.

### GET /health

Проверка состояния сервера и загрузки модели.

Пример запроса через curl:

```
curl http://localhost:8000/health
```

Пример ответа:

```json
{
  "status": "работает",
  "model_loaded": true
}
```

- `model_loaded: true` — модель загружена, можно делать предсказания
- `model_loaded: false` — модель не загружена, сначала загрузите её

### POST /model/upload-model

Загружает модель в формате pickle (.pkl) на сервер.

Пример запроса через curl:

```
curl -X POST -F "file=@путь/к/файлу/model.pkl" http://localhost:8000/model/upload-model
```

Пример ответа:

```json
{
  "message": "Модель успешно загружена",
  "filename": "model.pkl"
}
```

### POST /model/predict

Предсказание по JSON с одним или несколькими клиентами.

Тело запроса должно содержать поле `records` со списком объектов. Каждый объект — данные одного клиента.

Пример запроса через curl:

```
curl -X POST http://localhost:8000/model/predict \
  -H "Content-Type: application/json" \
  -d '{
    "records": [{
      "person_age": 25,
      "person_gender": 1,
      "person_education": 2,
      "person_income": 50000,
      "person_emp_exp": 5,
      "person_home_ownership": 1,
      "loan_amnt": 150000,
      "loan_intent": 0,
      "loan_int_rate": 6.5,
      "loan_percent_income": 0.3,
      "cb_person_cred_hist_length": 5,
      "credit_score": 720,
      "previous_loan_defaults_on_file": 1
    }]
  }'
```

Пример ответа:

```json
{
  "records": [
    {
      "loan_status": 1,
      "loan_status_label": "Одобрено",
      "probability": 0.85
    }
  ],
  "count": 1
}
```

- `loan_status`: 1 — одобрено, 0 — отказ
- `loan_status_label`: текстовая расшифровка
- `probability`: уверенность модели от 0 до 1 (чем ближе к 1, тем более вероятно одобрение)

### POST /model/predict-from-csv

Предсказание из CSV-файла. Файл передаётся как form-data с ключом `file`.

Пример запроса через curl:

```
curl -X POST -F "file=@путь/к/файлу/clients.csv" http://localhost:8000/model/predict-from-csv
```

Пример ответа:

```json
{
  "message": "Предсказание выполнено",
  "rows_processed": 100,
  "csv_data": "person_age,person_gender,...,prediction\n25,1,...,1\n30,0,...,0\n...",
  "roc_auc": 0.92
}
```

- `csv_data`: строка с CSV, где к исходным данным добавлена колонка `prediction`
- `roc_auc`: метрика качества, если в CSV был столбец с правильными ответами

### GET /model/stats

Статистика использования модели.

Пример запроса:

```
curl http://localhost:8000/model/stats
```

Пример ответа:

```json
{
  "data": {
    "models_uploaded": 2,
    "predictions_made": 150,
    "approved": 95,
    "rejected": 55,
    "recent_predictions": [ ... ]
  }
}
```

### Поля для одного клиента

| Поле | Тип | Описание | Допустимые значения |
|------|-----|----------|-------------------|
| person_age | число | Возраст клиента | от 18 до 100 |
| person_gender | число | Пол | 0 — женский, 1 — мужской |
| person_education | число | Уровень образования | 0 — общее, 1 — среднее профессиональное, 2 — высшее |
| person_income | число | Годовой доход | больше 0 |
| person_emp_exp | число | Стаж работы в годах | от 0 до 50 |
| person_home_ownership | число | Тип жилья | 0 — арендованное, 1 — собственное, 2 — ипотечное |
| loan_amnt | число | Сумма кредита | больше 0 |

## Как всё устроено внутри

### Серверная часть (бэкенд)

Файл `app/main.py` — точка входа. Он создаёт приложение FastAPI и подключает:

- Папку `frontend/` как статические файлы, чтобы браузер мог загружать HTML, CSS и JS
- Роутер `model.py`, который содержит эндпоинты `/model/upload-model`, `/model/predict`, `/model/predict-from-csv`, `/model/stats`
- Два простых эндпоинта: `/` (главная страница с информацией о сервисе) и `/health` (проверка состояния)

Файл `app/routers/model.py` содержит обработчики:

- `upload_model()` — принимает файл .pkl, сохраняет его и загружает модель в память
- `predict()` — принимает JSON с записями, отдаёт результат с предсказаниями
- `predict_from_csv()` — принимает CSV-файл, обрабатывает все строки, возвращает CSV с добавленной колонкой prediction
- `get_stats()` — возвращает статистику использования

Файл `app/services/predictor.py` содержит класс `ModelService`:

- `load_model()` — загружает pickle-файл и извлекает из него модель и названия признаков
- `predict()` — вызывает метод `predict_proba()` у модели, форматирует результат
- Хранит состояние: загружена ли модель, сколько было предсказаний

### Клиентская часть (фронтенд)

Файл `frontend/index.html` — структура страницы: шапка, форма ввода, таблица записей, кнопки отправки, секция CSV, секция статистики.

Файл `frontend/style.css` — оформление: цвета, шрифты, отступы, сетка формы, стили таблиц, стили для результатов (зелёный для одобрения, красный для отказа).

Файл `frontend/script.js` — вся интерактивность:

- Собирает данные из формы и добавляет их в таблицу
- Валидирует (проверяет) данные перед отправкой
- Отправляет POST-запросы на `/model/predict` и `/model/predict-from-csv`
- Запрашивает статистику с `/model/stats`
- Проверяет состояние сервера каждые 15 секунд через `/health`
- Обновляет индикатор статуса (зелёная точка — модель загружена, красная — сервер недоступен)

### Как загружается и работает модель

1. Модель хранится в файле `.pkl` (формат pickle — встроенный сериализатор Python)
2. Внутри pickle-файла должен быть словарь (dict) с ключами:
   - `"model"` — объект обученной модели scikit-learn
   - `"feature_names"` — список из 13 названий признаков в правильном порядке
3. Когда пользователь загружает файл через веб-интерфейс или API, сервер вызывает `pickle.load()`, извлекает модель и запоминает её
4. При запросе на предсказание сервер:
   - Берёт входные данные (JSON или CSV)
   - Преобразует их в массив numpy
   - Вызывает `model.predict_proba()`, чтобы получить вероятности для класса "одобрено" (1)
   - Если вероятность выше 0.5 — одобрено, иначе — отказ
   - Формирует ответ и возвращает его
